    * ```CMakeLists.txt```
    * ```lib.cpp```
    * ```lib.h```
  * ```C/```
    * ```CMakeLists.txt```
    * ```main_test.cpp```
    * ```main_test.h```
    * ```main.cpp```
    * ```main.h```
    * ```task_A.cpp```
    * ```task_A.h```
    * ```task_B.cpp```
    * ```task_B.h```
  * ***```bin/```***
    * ***```C```***
    * ***```CTest```***
  * ***```build/```***
  * ***```lib/```***
  * ```README.MD```
  * ```CMakeLists.txt```

## Сборка проекта

```sh
cd task-cmake
mkdir build
cd build
cmake ..
make
cd ..
```

Или одной командой:

```sh
cd task-cmake && mkdir build && cd build && cmake .. && make && cd ..
```

## Описание каждого файла

### ```A/preparing.py```

Python-скрипт, который генерирует index.h

### ```A/index.h```

Header-файл, который генерируется после запуска сборки проекта. При каждой сборке содержимое перезаписывается файлом из ```A/preparing.py```.\
Содержит реализацию функции ```int GetIndex(const std::vector<int>, int)```.\
Получение индекса элемента в массиве за линейное время.\
Если элементов несколько, то возвращается индекс первого такого элемента.\
Если элемента в векторе нет, то возвращается -1.

### ```B/CMakeLists.txt```

CMake-файл, который помогает собрать библиотеку сортировки. Запускается автоматически при сборке главного CMake-файла.

### ```B/lib.cpp```, ```B/lib.h```

 Реализация (в cpp-файле) и объявление (в хедере) функции сортировки выбором за линейное время. Ничего не возвращает, принимает ```std::vector<int>``` по ссылке.

### ```C/CMakeLists.txt```

CMake-файл, для тестирования файлов.\
Инструмент: GoogleTest.

### ```C/main_test.cpp```, ```C/main_test.h```

Запуск всех тестов, которые расположены в файлах ```task_A.cpp```, ```task_B.cpp```.\
Комплируется в ```bin/CTest```.

### ```C/main.cpp```, ```C/main.h```

Пример работы функции сортировки вектора. Пользователь вводит количество элементов и сами элементов.\
Компилируется в ```bin/C```.

### ```C/task_A.cpp```, ```C/task_A.h```

Тестирование функции получение индекса (```A/index.h```).

### ```C/task_B.cpp```, ```C/task_B.h```

Тестирование функции сортировки массива (```B/lib.cpp```).

### ```CMakeLists.txt```

Главный CMake-файл, который запускает дочерние CMake-файлы (```A/CMakeLists.txt```, ```B/CMakeLists.txt```).
В итоге сборки:

1. Собирается библиотека сортировки в ```lib/```
2. Собираются исполняемые файлы ```bin/C```, ```bin/CTest```
